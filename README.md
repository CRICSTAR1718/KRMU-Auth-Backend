# Auth Backend - Complete Implementation

A full-stack Express.js REST API with JWT cookie-based authentication, Drizzle ORM, Zod validation, and Supabase PostgreSQL.

## Project Structure

```
Auth-Backend/
├── .env                          # Environment variables (create with Supabase URL)
├── .gitignore                    # Git ignore rules
├── package.json                  # Dependencies
├── drizzle.config.js             # Drizzle-kit configuration
├── app.js                        # Express app entry point
├── config/
│   └── db-config.js              # Drizzle ORM + Postgres connection
├── schema/
│   ├── user.model.js             # Users table schema
│   └── product.model.js          # Products table schema
├── middleware/

├── api/
│   ├── auth/
│   │   ├── auth.router.js        # Auth endpoints
│   │   ├── auth.controllers.js   # Register, login, logout logic
│   │   └── dto/
│   │       ├── register.dto.js   # Registration validation (Zod)
│   │       ├── login.dto.js      # Login validation (Zod)
│   │       └── logout.dto.js     # Logout validation (cookie check)

```

## Setup Instructions

### 1. Install Dependencies

```bash
npm install
```

### 2. Configure Environment

Create/update `.env` with your Supabase credentials:

```env
PORT=6101
DATABASE_URL=postgresql://postgres.[your-ref]:[your-password]@aws-0-[your-region].pooler.supabase.com:6543/postgres
JWT_SECRET=your_super_secret_jwt_key_here_change_in_production
ALLOWED_ORIGIN=http://localhost:3000
```

**Note:** Use the **pooler** URL from Supabase (port 6543), not the direct connection.

### 3. Generate & Push Schema

```bash
# Generate migrations from schema
npm run db:generate

# Push schema to Supabase (creates tables)
npm run db:push
```

Alternatively, to migrate step-by-step:

```bash
npm run db:migrate
```

### 4. Start Development Server

```bash
npm run dev
```

Server runs on `http://localhost:6101`

## API Documentation

### Health Check

```http
GET /health
```

**Response:**
```json
{ "success": true, "message": "Server is running" }
```

### Auth Endpoints

#### Register

```http
POST /api/v1/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securePassword123"
}
```

**Success (201):**
```json
{ "success": true, "message": "User registered successfully!" }
```

**Error (409 - User exists):**
```json
{ "success": false, "message": "User already exists!" }
```

#### Login

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securePassword123"
}
```

**Success (200):**
```json
{ "success": true, "message": "Login successful!" }
```
Sets httpOnly cookie: `token=<JWT>`

**Error (404 - User not found):**
```json
{ "success": false, "message": "User not found!" }
```

**Error (401 - Wrong password):**
```json
{ "success": false, "message": "Incorrect password!" }
```

#### Logout

```http
POST /api/v1/auth/logout
Cookie: token=<JWT>
```

**Success (200):**
```json
{ "success": true, "message": "Logged out successfully!" }
```

Clears the `token` cookie.

### Product Endpoints

#### Get All Products (Public)

```http
GET /api/v1/products
```

**Response (200):**
```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "name": "Product Name",
      "price": "99.99",
      "stock": 10,
      "images": ["url1", "url2"],
      "user_id": "uuid",
      "created_at": "2026-06-23T12:00:00Z",
      "updated_at": "2026-06-23T12:00:00Z"
    }
  ]
}
```

#### Create Product (Protected)

```http
POST /api/v1/products
Cookie: token=<JWT>
Content-Type: application/json

{
  "name": "New Product",
  "price": 49.99,
  "stock": 5,
  "images": ["https://example.com/img1.jpg"]
}
```

**Success (201):**
```json
{
  "success": true,
  "data": { /* product object */ }
}
```

#### Update Product (Protected)

```http
PATCH /api/v1/products/:productId
Cookie: token=<JWT>
Content-Type: application/json

{
  "name": "Updated Name",
  "price": 59.99
}
```

**Success (200):**
```json
{
  "success": true,
  "data": { /* updated product object */ }
}
```

**Error (403):**
```json
{ "success": false, "message": "Forbidden" }
```
(Only the creator can update their products)

## Database Schema

### Users Table

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | UUID | PK, default `gen_random_uuid()` |
| `email` | TEXT | UNIQUE, NOT NULL |
| `password` | TEXT | NOT NULL (bcrypt hash) |
| `name` | TEXT | NULL |
| `created_at` | TIMESTAMP | NOT NULL, default `now()` |
| `updated_at` | TIMESTAMP | NOT NULL, default `now()` |

### Products Table

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | UUID | PK, default `gen_random_uuid()` |
| `name` | TEXT | NOT NULL |
| `price` | NUMERIC | NOT NULL |
| `stock` | INTEGER | NOT NULL |
| `images` | JSONB | NOT NULL, default `[]` |
| `user_id` | UUID | NOT NULL, FK → users(id) ON DELETE CASCADE |
| `created_at` | TIMESTAMP | NOT NULL, default `now()` |
| `updated_at` | TIMESTAMP | NOT NULL, default `now()` |

## Authentication Flow

1. **Register**: User provides email + password → hashed with bcrypt (10 rounds) → stored in DB
2. **Login**: User provides email + password → bcrypt compare → JWT signed (7d expiry) → set httpOnly cookie
3. **Protected Routes**: JWT verified from cookie → decoded payload attached to `req.user`
4. **Logout**: Cookie cleared → session ends

### Security Features

- ✅ **httpOnly cookies** - prevents XSS attacks
- ✅ **secure flag** - cookies only sent over HTTPS
- ✅ **sameSite: strict** - prevents CSRF attacks
- ✅ **bcrypt hashing** (10 rounds) - passwords never stored in plain text
- ✅ **JWT expiry** (7 days) - limits token lifetime
- ✅ **CORS** with credentials - controlled cross-origin access
- ✅ **Zod validation** - input sanitization on all endpoints

## Common Issues & Troubleshooting

### Issue: "connect ECONNREFUSED"

**Cause:** Database connection failed  
**Solution:** Verify `DATABASE_URL` in `.env` is correct and Supabase is running

### Issue: "401 Unauthorized"

**Cause:** Missing or invalid JWT cookie  
**Solution:** Login first to get a valid token, or check cookie is being sent with requests

### Issue: "403 Forbidden" on PATCH /products/:id

**Cause:** Trying to update another user's product  
**Solution:** Only the creator of a product can update it

### Issue: "prepare: false" errors

**Cause:** Using direct Postgres connection with pooler  
**Solution:** Ensure `prepare: false` is set in db-config.js (already configured)

## Testing with curl

### Register
```bash
curl -X POST http://localhost:6101/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'
```

```bash
curl -X POST http://localhost:6101/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}' \
  -c cookies.txt
```

### Get Products
```bash
curl http://localhost:6101/api/v1/products
```

### Create Product (with auth cookie)
```bash
curl -X POST http://localhost:6101/api/v1/products \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"name":"Laptop","price":999.99,"stock":5}'
```

### Logout
```bash
curl -X POST http://localhost:6101/api/v1/auth/logout \
  -b cookies.txt
```
